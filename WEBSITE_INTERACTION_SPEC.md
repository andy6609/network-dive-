# 웹사이트 인터랙션 스펙 — Network Dive

> 스크롤/클릭에 따라 히어로 영상이 어떻게 동작하는지 정의하는 **작동 방식 명세**.
> 이 흐름은 유저(발표 청중/포트폴리오 방문자)가 직접 만지는 경험을 설계한 것. 창의 방향은 프로젝트 오너가 정의함.
> 관련 문서: [PROJECT_STATUS.md](PROJECT_STATUS.md) (전체 진행), [main.purpose.md](main.purpose.md) (프로젝트 배경)

최종 수정: 2026-07-08

## 1. 재생 방식 4가지 (용어 정의)

이 사이트는 단일 스크럽이 아니라 **4가지 재생 방식이 구간마다 전환**되는 하이브리드다.

| 방식 | 동작 | 역재생 | 구현 개념 |
|---|---|---|---|
| **A. 프레임 스크럽** | 스크롤 위치 = 영상의 프레임 번호. 스크롤한 만큼, 스크롤 방향대로 | ✅ 됨 | video-to-website 스킬 (FFmpeg 프레임 + ScrollTrigger scrub) |
| **B. 자동 애니메이션** | 트리거되면 앞으로 재생. **재생 동안 페이지 스크롤 고정(pin)**, 끝나면 스크롤 해제 | ❌ 안 됨 | ScrollTrigger pin + 타임라인/영상 자동재생 |
| **C. 연속 루프** | 배경에서 계속 반복 재생. 페이지는 그 위/아래로 일반 스크롤 | — | pin된 루프 배경 + 위에 콘텐츠 |
| **D. 스크롤 리빌** | 일반 웹페이지처럼 스크롤, 섹션이 페이드/슬라이드로 등장 | ✅ 됨 | 표준 스크롤 애니메이션 |

## 2. 전체 시퀀스 (구간별)

| # | 구간 | 화면에서 벌어지는 일 | 방식 | 역재생 |
|---|---|---|---|---|
| 1 | **KF0 (클릭 전)** | 스크롤할수록 전송(업로드) 버튼으로 줌인. 과하게 확대되지 않게 상한. 화면에 **"click to upload" 힌트** 표시 | A 스크럽 | ✅ |
| 2 | **[버튼 클릭]** | 유저가 업로드 버튼을 **실제로 클릭** → 이후 시퀀스 트리거 | 인터랙션 | — |
| 3 | KF0 → KF0.5 | 버튼 아래 **업로드 바가 띠리리 차오름** | B 자동애니 | ❌ |
| 4 | KF0.5 → KF1 | 유리 뚫고 화면 속으로, 파일이 0101 바이트로 분해 | B 자동애니 | ❌ |
| 5 | KF1 → KF2-1 | 바이트 스트림이 **위로 솟구침** → 카메라가 위로 이동해 내려다보는 각도로 재배치 → **KF2-1 도착 시 정지** | B 자동애니 | ❌ |
| 6 | **KF2-1** | (여기서 스크럽으로 전환) 카메라 각도가 **살짝 위 → 아래로 틸트**하며 폭포처럼 쏟아지는 0101을 따라 내려감 | A 스크럽 | ✅ |
| 7 | KF2-1 → KF2-2 | 계속 따라 내려가면 OSI **7계층**이 드러나고, 0101이 7층에서 1층으로 쏟아지는 걸 스크롤로 따라감 (눈에 보이는 속도) | A 스크럽 | ✅ |
| 8 | KF2-2 → KF3 | 케이블/라우터 도착. **애니로 하면 훅 지나가니까 스크럽으로 속도 조절점** 확보 | A 스크럽 | ✅ |
| 9 | KF3 → KF4 | 케이블이 중앙으로 모임 → **내가 0101 바이트 스트림이 되어 속도감 있게 중앙으로 타고 들어가는** 느낌 | B 자동애니 | ❌ |
| 10 | **KF4 (프록시)** | 바이트 스트림 좌→우로 흐르고 게이트 안 **스캔이 계속** 돌아감. 뒤 배경에 **"프록시가 무엇을 하는가" 설명 텍스트** 몇 개. 페이지는 이 루프 위에서 아래로 스크롤 | C 연속루프 | — |
| 11 | KF4 → KF5 | (KF4~KF5 사이에 **텍스트 섹션 몇 개** 검토 중) 스크롤 내리면 **file detected된 디지털 파일이 소환되듯** 자연스럽게 등장 | D 리빌 | ✅ |
| 12 | KF5 → KF6 | 스크롤 → 전환되며 **KF6(관제 센터 경고 대시보드)** 등장 | D 리빌 | ✅ |

**흐름 요약:** 스크럽(A) → [클릭] → 자동애니(B) → 스크럽(A, KF2-1~KF3) → 자동애니(B) → 루프(C) → 리빌(D)

## 3. 구간별 상세 노트

### KF0 — 줌인 + 클릭 게이트
- 스크롤 = 전송 버튼으로 줌인. **줌 상한**을 둬서 눈에 부담되게 과확대되지 않도록.
- 클릭 전까지는 **역재생 가능** (스크롤 위로 = 줌아웃).
- **버튼은 실제 클릭 필요.** 유저가 놓치지 않게 **"click to upload" 힌트**(텍스트/커서 유도)를 반드시 노출.
- 클릭하는 순간 = 시퀀스의 방아쇠. 이 전과 후로 사이트 상태가 갈림 (before-click / after-click).

### KF0.5 → KF1 → KF2-1 — 자동 다이브 (핀 걸고 재생)
- 클릭 직후 **업로드 바 차오름** → 유리 관통 → 0101 분해 → 스트림 위로 솟구침, 여기까지 **한 번에 자동 재생**.
- 재생 동안 **페이지 스크롤 고정(pin)**. 유저가 스크롤해도 이 애니는 앞으로만, 역재생 없음.
- **KF1 방향성:** 바이트 스트림이 **위로 솟음** → 카메라가 위로 올라가 **내려다보는 각도로 재배치** → 그래야 KF2의 "아래로 쏟아지는 폭포"로 자연 연결. (위로 솟았다가 위에서 내려다보며 아래 낙하를 따라가는 흐름)
- **KF2-1 도착 시 자동 재생 정지** → 여기서 스크럽 모드로 핸드오프.

### KF2-1 → KF2-2 → KF3 — 스크럽 하강 (하나의 연속 스크럽 구간)
- **카메라 각도 이동 필수:** KF2-1은 살짝 위에서 보는 각도 → 스크롤하면 **아래 각도로 틸트**하며 0101 폭포를 따라 내려감.
- 계속 내려가면 **KF2-2의 OSI 7계층**이 드러나고, 7층→1층으로 바이트 스트림이 쏟아지는 걸 스크롤로 따라감.
- **KF2-2 → KF3도 스크럽** — 케이블/라우터 도착을 유저가 속도 조절하며 넘어가게. (애니로 하면 훅 지나가 버림)
- 이 구간 전체 **역재생 가능**.

### KF3 → KF4 — 속도감 다이브 (자동애니)
- KF3은 케이블들이 화면 중앙으로 모이는 구도.
- **내가 0101 바이트 스트림이 되어 중앙 소실점으로 빨려 들어가는** 속도감 있는 자동 재생.

### KF4 — 프록시 루프 (연속 루프 + 텍스트)
- 게이트 안 스트림 좌→우, **스캔 계속** 반복 재생.
- **뒤 배경 텍스트:** "프록시가 무엇을 하는가" 설명 문구 몇 개 (예: 트래픽 가로채기 / 응용계층 해석 / 파일 업로드 식별).
- 페이지는 이 루프 위에서 **일반 스크롤로 아래로** 진행.

### KF4 → KF5 → KF6 — 리빌 (일반 스크롤)
- **KF4~KF5 사이에 텍스트 섹션 몇 개** 삽입 검토 중 (내용 미정).
- 스크롤 내리면 **KF5(file detected 디지털 파일)** 가 소환되듯 등장.
- 계속 스크롤 → **KF6(관제 센터 경고 대시보드)** 로 전환.
- 이 구간 역재생 가능(일반 스크롤).

## 4. 기술 구현 노트

- **상태 기계 필요:** "지금 어느 방식(A/B/C/D)인지"를 관리하는 컨트롤러를 얹어야 함. 유튜버의 단순 스크럽 스킬 하나로는 부족.
- **자동애니 = pin-and-play:** ScrollTrigger로 해당 섹션에 도달하면 스크롤을 pin(고정)하고 타임라인/영상을 자동재생, 완료 후 pin 해제하고 다음 구간으로. 이게 "역재생 방지 + 자동 진행"을 만드는 방법.
- **클릭 게이트:** KF0의 업로드 버튼은 실제 클릭 이벤트. 클릭 전에는 스크롤이 KF0 스크럽만 제어, 클릭 후 시퀀스 언락. (스크롤 진행을 클릭 전까지 KF0 구간에 가두는 처리 필요)
- **스크럽↔애니 핸드오프:** 자동애니가 끝나는 프레임 = 다음 스크럽 구간의 시작 프레임이 정확히 일치해야 점프 없음. (KF 이미지 연속성 락과 동일 원리)
- **에셋:** 각 구간 영상은 기존 clip 프롬프트(KFn→KFn+1) 기반. 루프인 KF4는 시작=끝 프레임이 같은 별도 루프 클립으로 뽑는 게 좋음.
